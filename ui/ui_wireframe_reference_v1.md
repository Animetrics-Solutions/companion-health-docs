# Companion Health — System Wireframe (Reference)
Version: v1.0
Status: ENFORCED

---

# 1. CORE MODEL

Single persistent canvas.

No screens.
No steps.
No navigation flow.

System evolves in place.

---

# 2. GRID SYSTEM

- 12 column layout
- Max content width: 560px (center column)
- Outer margins flexible but symmetrical

---

## Vertical Rhythm

Spacing ONLY:
- 24px
- 40px
- 64px

---

# 3. TOP BAR

Height: 64px

LEFT:
[Companion Health Logo]

RIGHT:
[● System Live]   [System]

---

## Rules

- No extra text
- No decorative elements
- Logo = full wordmark only

---

# 4. MAIN CANVAS (CENTER COLUMN)

Everything aligns to this column.

---

## STRUCTURE

[ PET META ]
[ RISK ]
[ STATUS ]
[ SIGNALS ]
[ PERSPECTIVE ]
[ CONTEXT LAYER ]
[ SYSTEM ACCESS ]

---

# 5. PET META

Small, low emphasis

Example:

BRUNO · AGE 11 · DOMESTIC SHORTHAIR

---

## Rules

- Uppercase
- Low opacity
- Not visually dominant

---

# 6. RISK (PRIMARY ELEMENT)

CENTERED

Example:

72

---

## Rules

- Largest element on screen
- Suisse Intl Medium
- No decoration
- No shadow

---

## Motion

- Pulse once on load (600ms)
- Subtle settle

---

# 7. STATUS

Below risk

Example:

MONITORED · ELEVATED

---

## Rules

- Small caps feel
- Slight letter spacing
- Muted color (amber for elevated)

---

# 8. SIGNALS

Below status

Example:

Hydration ↓  
Weight ↓  
Activity stable  

---

## Rules

- Inline or stacked
- No separators (avoid excessive dots)
- Left aligned within center column

---

# 9. PERSPECTIVE SWITCH

Text-based, NOT buttons

Example:

Pet Parent   Insurance   Veterinary

---

## Active State

- subtle underline OR
- slight brightness increase

---

## Rules

- No filled buttons
- No heavy borders

---

# 10. CONTEXT LAYER

Changes based on perspective

---

## PET PARENT

- simple observational statement
- no emotional exaggeration

---

## INSURANCE

3 metrics aligned in grid:

Risk Score  
Confidence  
Lead Time  

---

## VETERINARY

- short paragraph
- intervention window
- timeline (minimal)

---

## Rules

- No cards
- No boxes
- No shadows

---

# 11. SYSTEM ACCESS

Bottom right (or subtle position)

[View System →]

---

## Rules

- low emphasis
- not primary CTA

---

# 12. BACKGROUND SYSTEM

Signal field:

- very low opacity
- slow drift
- minimal density

---

## Rules

- never dominant
- never decorative

---

# 13. MOTION SYSTEM

Allowed:

- pulse (risk)
- drift (signals)
- emerge (data)
- tick (updates)

---

Forbidden:

- page transitions
- slides
- bounces
- loops without cause

---

# 14. COLOR SYSTEM

Primary:

- neutral (white/grey)

Accent:

- green → active state only
- amber → elevated risk

---

## Rules

- no color decoration
- color = meaning only

---

# 15. WHAT MUST NOT EXIST

- cards
- dashboards
- panels (unless system layer)
- onboarding UI
- step indicators
- next buttons

---

# 16. EXPERIENCE CHECK

If user sees this:

They should feel:

“This system is already running”

NOT:

“This is a UI being shown to me”

---

# FINAL RULE

Do not design screens.

Design system state.
