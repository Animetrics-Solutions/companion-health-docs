Design Principles — Companion Health
Classification: INTERNAL — CONFIDENTIAL
Version: v1.0
Status: ENFORCED

1. PURPOSE

Defines the visual philosophy and constraints for all Companion Health interfaces.

This ensures:

Consistency
Clarity
High-trust perception
2. DESIGN POSITIONING

Companion Health is not a marketing product.

It is:

A clinical intelligence system

The interface must reflect:

Precision
Calm authority
Technical credibility
3. CORE PRINCIPLES
3.1 Visual Hierarchy (Single Focus Rule)

Each screen must have:

One dominant element
Supporting secondary elements

Avoid:

Equal visual weight across sections
3.2 Whitespace Discipline

Whitespace is a primary design tool.

Rules:

Increase vertical spacing between sections
Avoid dense layouts
Allow elements to “breathe”
3.3 Minimal Color Usage

Color is functional, not decorative.

Use color ONLY for:

Risk states (green / yellow / red)
Primary actions

Everything else:

Neutral palette
3.4 No UI Noise

Avoid:

Unnecessary borders
Card-heavy layouts
Decorative elements

Use:

Spacing instead of containers
3.5 Grid Integrity

All elements must align to a structured grid.

Grid usage:

Subtle
Background-level
Never dominant
3.6 Typography Discipline
Strong hierarchy (headline vs body)
Minimal font variation
Clear readability
3.7 Information Compression

Interfaces must be scannable within 3–5 seconds.

Remove:

Redundant text
Long explanations
3.8 Motion Philosophy
Subtle (150–300ms)
Functional only
No expressive animations
4. COMPONENT BEHAVIOUR
4.1 Signal Elements
Must be immediately interpretable
Color-coded only when necessary
4.2 Data Blocks (API / JSON)
Clean
Structured
Minimal styling
4.3 Layout Flow

Flow must feel:

System → Signal → Risk → Decision

5. DESIGN ANTI-PATTERNS

Do NOT introduce:

Marketing-style sections
Feature overload
Bright or saturated palettes
Complex UI interactions
6. QUALITY STANDARD

All designs must feel:

Intentional
Minimal
System-driven

If a design feels:

crowded
decorative
“startup-like”

It must be reduced.

7. ENFORCEMENT RULE

Before finalizing any UI:

Ask:

Can this be simplified further?
Is every element necessary?
Does this increase clarity?

If not → remove
