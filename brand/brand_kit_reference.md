# Companion Health — Brand Kit Reference
Version: v1.0
Status: ENFORCED SYSTEM

---

# 1. BRAND ESSENCE

Companion Health is not a lifestyle brand.

It represents:
- intelligence
- system thinking
- quiet authority
- clinical precision

---

## Tone

- Calm
- Controlled
- Observational
- Non-emotional

---

## Avoid

- hype language
- marketing exaggeration
- emotional persuasion

---

# 2. BRAND ARCHITECTURE

## 2.1 Companion Health (Primary Identity)

Represents:
- intelligence layer
- platform
- system

Used for:
- product UI
- reports
- homepage
- investor material

---

## 2.2 Fetch™ API (Product Layer)

Represents:
- API layer
- developer interface
- integration system

Used for:
- API docs
- dev portal
- integration surfaces

---

## Rule

Fetch NEVER replaces Companion.

Hierarchy:

Companion Health → Fetch API

---

# 3. LOGO SYSTEM

## 3.1 Logo Types

### A. Full Logo (Primary)

Use for:
- navbar
- reports
- presentations
- homepage

---

### B. Brandmark (Symbol Only)

Use for:
- favicon
- subtle UI presence
- watermark (low opacity)
- system indicator

---

### C. Fetch Logo

Use only in:
- API sections
- developer documentation

---

## 3.2 Usage Rules

### Navbar

- ONLY full Companion logo
- no text recreation
- no brandmark + text combo

---

### Footer

- brandmark allowed
- optional full logo

---

### UI

- brandmark only if subtle
- never dominant

---

## 3.3 What NOT to do

- stretch logo
- recolor arbitrarily
- add shadows
- place on noisy backgrounds
- rotate or animate logo

---

# 4. TYPOGRAPHY SYSTEM

## 4.1 Primary Fonts

### Faire Octave

Use ONLY for:
- hero headlines
- section anchors (limited)

Tone:
- editorial authority

---

### Suisse Intl

Use for:
- all UI
- body text
- system labels
- data

Tone:
- precision
- clarity

---

### Suisse Intl Mono

Use for:
- API labels
- code
- metadata
- grid references

Tone:
- technical layer

---

## 4.2 Hierarchy

### H1

- Faire Octave
- large
- tight

---

### H2

- Faire Octave (limited use)

---

### Body

- Suisse Intl

---

### Labels / Data

- Suisse Intl / Mono

---

## 4.3 Rules

- No additional fonts
- No decorative typography
- No excessive weight variation

---

# 6. VISUAL LANGUAGE

## 6.1 System Aesthetic

- minimal
- restrained
- precise

---

## 6.2 Background System

Use:
- soft gradients
- signal fields
- low opacity patterns

Purpose:
- suggest system activity
- not decoration

---

## 6.3 Graphic Elements

Use sparingly:

- radial fades
- signal clusters
- subtle motion layers

---

## Avoid

- illustrations
- decorative shapes
- visual noise

---

# 7. GRID SYSTEM

## Structure

- 12 column grid
- center column preferred (560px content)

---

## Spacing

Use ONLY:

- 24px
- 40px
- 64px

---

## Rules

- everything aligned
- no floating elements
- no inconsistent spacing

---

# 8. MOTION PRINCIPLES

## Allowed

- subtle drift
- pulse (single)
- fade transitions
- data tick

---

## Not Allowed

- bouncing
- looping without cause
- flashy transitions
- animation for decoration

---

## Rule

Motion must answer:

“What changed?”

---

# 9. UI PRINCIPLES

## System First

UI must feel:

- live
- continuous
- deterministic

---

## Avoid

- dashboards
- cards
- heavy containers
- onboarding flows

---

## Prefer

- spacing
- alignment
- hierarchy

---

# 10. CONTENT STYLE

## Writing Tone

- observational
- factual
- restrained

---

## Example

Instead of:
“Your pet needs urgent care!”

Use:
“Signals indicate elevated risk”

---

## Rule

Explain less.
Show more.

---

# 11. REPORT VS PRODUCT

## Reports

- serif-led (controlled)
- structured
- dense

---

## Product UI

- sans-led
- minimal
- real-time feel

---

## Rule

Never mix the two styles

---

# 12. FINAL PRINCIPLE

Companion Health is not expressive.

It is:

A system that reveals.

---

# FINAL RULE

If it looks designed → reduce

If it feels inevitable → correct
